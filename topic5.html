HTML Media Tags – Video and Audio Guide

This page explains how to use HTML media tags such as <video> and <audio>. It covers basic usage, attributes, and advanced features to help you include multimedia content in your web pages.

1. What are Media Tags?

Media tags allow you to embed audio and video files directly into web pages without requiring external plugins. The two main tags are:

  • <video> – For embedding video files.
  • <audio> – For embedding audio files.

2. The <video> Tag

The <video> tag is used to embed video content. It supports multiple formats like MP4, WebM, and Ogg.

2.1 Basic Example

<video width="320" height="240" controls>
    <source src="movie.mp4" type="video/mp4">
    Your browser does not support the video tag.
</video>

Explanation:

  • width and height: Set the video size.
  • controls: Adds play, pause, volume, etc.
  • <source>: Specifies the video file and format.
  • The fallback text is shown if the browser doesn't support videos.

2.2 Multiple Sources

Provide multiple formats for better compatibility.

<video width="320" height="240" controls>
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.webm" type="video/webm">
    <source src="movie.ogv" type="video/ogg">
    Your browser does not support the video tag.
</video>

2.3 Advanced Attributes

  • autoplay: Video starts playing automatically.
  • loop: Repeats the video.
  • muted: Starts without sound.
  • poster: Shows an image before the video plays.
<video width="320" height="240" controls autoplay loop muted poster="thumbnail.jpg">
    <source src="movie.mp4" type="video/mp4">
    Your browser does not support the video tag.
</video>

3. The <audio> Tag

The <audio> tag is used to embed audio files. It supports formats like MP3, WAV, and Ogg.

3.1 Basic Example

<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    Your browser does not support the audio tag.
</audio>
  • controls: Adds play, pause, volume controls.
  • <source>: Specifies the audio file and format.

3.2 Multiple Sources

To ensure compatibility, provide multiple formats:

<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    Your browser does not support the audio tag.
</audio>

3.3 Advanced Attributes

  • autoplay: Audio starts playing automatically.
  • loop: Audio repeats continuously.
  • muted: Starts without sound.
  • preload: Specifies how much of the file to load before playing.
<audio controls autoplay loop muted preload="auto">
    <source src="audio.mp3" type="audio/mpeg">
    Your browser does not support the audio tag.
</audio>

4. Attributes Common to Both Media Tags

  • controls: Show playback controls.
  • autoplay: Start automatically on page load.
  • loop: Repeat indefinitely.
  • muted: Start without sound.
  • preload: Defines how the file is loaded before playback (auto, metadata, none).
  • poster (video only): Image to show before playback.

5. Using Captions and Subtitles

You can add captions or subtitles using the <track> tag inside the <video> tag. 

<video width="320" height="240" controls>
    <source src="movie.mp4" type="video/mp4">
    <track src="subtitles_en.vtt" kind="subtitles" srclang="en" label="English">
    Your browser does not support the video tag.
</video>
  • kind: Defines the type like subtitles or captions.
  • srclang: Language of the track.
  • label: Name shown in the player settings.

6. Accessibility Considerations

  • Always include fallback text for unsupported browsers.
  • Use captions and subtitles to support different audiences.
  • Provide alternative text like <track> or descriptive content.

7. Example – Complete Media Section

<h2>Watch Our Video</h2>
<video width="480" controls poster="thumbnail.jpg">
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.webm" type="video/webm">
    <track src="subtitles_en.vtt" kind="subtitles" srclang="en" label="English">
    Your browser does not support the video tag.
</video>

<h2>Listen to Our Audio</h2>
<audio controls preload="auto">
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    Your browser does not support the audio tag.
</audio>

8. Summary

  • <video> and <audio> tags embed media files directly into HTML pages.
  • Multiple formats ensure compatibility across browsers.
  • Controls, autoplay, loop, and muted attributes enhance user experience.
  • Captions and subtitles improve accessibility.
  • Fallback content ensures graceful degradation when media is not supported.

This tutorial gives you everything needed to integrate video and audio into your web pages, from simple playback to advanced features like subtitles and accessibility support.
